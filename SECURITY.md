# Security Policy

## Supported versions

The latest `1.x` release receives security fixes. Users should reproduce a
reported issue against the newest release before reporting it when doing so is
safe.

## Reporting a vulnerability

Do not open a public Issue for a suspected vulnerability.

Use GitHub's **Report a vulnerability** entry in the repository Security tab to
send a private report. Include the affected version, impact, reproduction
steps, and any suggested mitigation. Remove credentials, personal data, and
unrelated production information from the report.

The maintainers will acknowledge the report, assess severity and scope, and
coordinate a fix and disclosure timeline. Please allow a reasonable remediation
window before public disclosure.

Security reports should focus on oh-my-multica itself. Vulnerabilities in
Multica, a Coding Agent runtime, GitHub, or another integration should also be
reported to that upstream project through its security process.
