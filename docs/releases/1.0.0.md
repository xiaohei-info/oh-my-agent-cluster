# oh-my-multica v1.0.0

oh-my-multica 1.0.0 is the first public release. It builds on Multica's shared
workspaces, work items, task queues, and Coding Agent runtimes, then adds the
delivery control needed to move a requirement through design, implementation,
review, merge, and final acceptance.

Agents decide how to design and decompose the work. A deterministic Loop owns
the parts that should not depend on an Agent's judgment: dependency scheduling,
evidence gates, bounded rework, recovery, merge conditions, and the final stop
decision.

## What you can do

- Generate reviewed design, acceptance criteria, project rules, and a dynamic
  manifest DAG for the current repository.
- Run independent tasks in parallel through Multica while respecting their
  dependencies.
- Require structured verification evidence and independent review before a
  node can advance.
- Connect CI and Pull Requests, merge eligible changes, and test the integrated
  default branch against flow-based acceptance criteria.
- Resume interrupted deliveries from persistent state instead of reconstructing
  progress from Agent conversations.
- Operate the same delivery protocol as a human, a Controller Agent, or through
  the local read-only web interface.
- Start from built-in Agent Team templates and choose different model tiers for
  planning, implementation, review, and acceptance.

## Before you start

- Python 3.10 or later.
- A Git repository with at least one commit and a pushable `origin` remote.
- Multica CLI and connected Coding Agent runtimes when using the Multica engine.

## See a complete delivery

The [Webhook Inbox demo] starts with a single production-constrained service
requirement. Planning Agents turn it into a five-node DAG, implementation Agents
work on independent nodes in parallel, and separate Reviewer and Acceptor Agents
verify the changes before and after merge.

The delivered FastAPI and SQLite service verifies signed webhooks, handles
concurrent retries without duplicate records, rejects conflicting content, and
runs in a non-root container. The repository includes the original requirement,
all five Pull Requests, 86 passing tests, 97.18% coverage, and 11/11 final
acceptance flows.

[Webhook Inbox demo]: https://github.com/xiaohei-info/oh-my-multica-demo-webhook-inbox#readme
